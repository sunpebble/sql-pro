# Query Library

Quarry keeps track of the queries you run and lets you save the ones you want to reuse. The Query Library combines **execution history** with **saved favorites**, and the **SQL Log** records every statement the app executes.

## Key Features

| Feature       | Description                                                    |
| ------------- | -------------------------------------------------------------- |
| **History**   | Recently executed queries, ready to load and rerun             |
| **Favorites** | Save queries you want to keep with <kbd>⌘</kbd> + <kbd>S</kbd> |
| **SQL Log**   | A complete log of SQL statements the app has executed          |

## Query History

Quarry automatically records the queries you execute. This makes it easy to revisit past work, rerun queries, and learn from previous attempts.

To reuse a query from history, open the Query Library, find the entry, and load it into the editor.

## Saving Favorites

To keep a query permanently:

1. Write the query in the SQL editor
2. Choose **Query → Save Query** or press <kbd>⌘</kbd> + <kbd>S</kbd>
3. The query is added to your library for quick reuse

Favorites are ideal for reports and maintenance statements you run regularly.

## SQL Log

The SQL Log shows every statement Quarry has executed against the database — including the statements generated by the UI (loading tables, applying edits, and so on). It's useful for:

- **Auditing** what the app actually ran
- **Debugging** unexpected results
- **Learning** what SQL corresponds to UI actions

## Keyboard Shortcuts

| Action     | Shortcut                    |
| ---------- | --------------------------- |
| Run query  | <kbd>⌘</kbd> + <kbd>R</kbd> |
| Save query | <kbd>⌘</kbd> + <kbd>S</kbd> |

## Tips and Best Practices

1. **Check history before rewriting** - The query you need may already be there
2. **Save favorites with intent** - Keep the library focused on queries you actually rerun
3. **Use the SQL Log when debugging** - See exactly what was sent to the database
