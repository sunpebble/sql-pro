## Vim Mode

SQL Pro supports optional Vim-style keybindings for power users. There are two separate Vim modes:

1. **Editor Vim Mode** - Vim keybindings in the SQL editor
2. **App Vim Mode** - Vim-style navigation throughout the app (sidebar, data grid)

Enable these in **Settings** > **Vim Settings**.

### Editor Vim Mode

When enabled, the SQL editor supports standard Vim keybindings:

| Mode   | Key                                                 | Action              |
| ------ | --------------------------------------------------- | ------------------- |
| Normal | <kbd>h</kbd> <kbd>j</kbd> <kbd>k</kbd> <kbd>l</kbd> | Move cursor         |
| Normal | <kbd>w</kbd> <kbd>b</kbd> <kbd>e</kbd>              | Word navigation     |
| Normal | <kbd>i</kbd>                                        | Enter insert mode   |
| Normal | <kbd>a</kbd>                                        | Append after cursor |
| Normal | <kbd>o</kbd>                                        | Open line below     |
| Normal | <kbd>O</kbd>                                        | Open line above     |
| Normal | <kbd>d</kbd><kbd>d</kbd>                            | Delete line         |
| Normal | <kbd>y</kbd><kbd>y</kbd>                            | Yank (copy) line    |
| Normal | <kbd>p</kbd>                                        | Paste               |
| Normal | <kbd>/</kbd>                                        | Search              |
| Insert | <kbd>Escape</kbd>                                   | Exit to normal mode |

::: tip Vim Status Bar
When Editor Vim Mode is enabled, a status bar appears below the editor showing the current mode (NORMAL, INSERT, VISUAL, etc.) and pending commands.
:::

### App Vim Mode (Sidebar & Data Grid)

When App Vim Mode is enabled, you can navigate the sidebar and data grid with Vim keys.

#### Navigation

| Key                      | Action               |
| ------------------------ | -------------------- |
| <kbd>j</kbd>             | Move down            |
| <kbd>k</kbd>             | Move up              |
| <kbd>h</kbd>             | Move left / Collapse |
| <kbd>l</kbd>             | Move right / Expand  |
| <kbd>g</kbd><kbd>g</kbd> | Jump to first item   |
| <kbd>G</kbd>             | Jump to last item    |
| <kbd>0</kbd>             | Jump to first column |
| <kbd>$</kbd>             | Jump to last column  |

#### Actions

| Key                                 | Action                                           |
| ----------------------------------- | ------------------------------------------------ |
| <kbd>i</kbd>                        | Enter edit mode (data grid)                      |
| <kbd>Enter</kbd>                    | Select item / Start editing                      |
| <kbd>Shift</kbd> + <kbd>Enter</kbd> | Enter edit mode                                  |
| <kbd>o</kbd>                        | Toggle expand (sidebar) / Insert row (data grid) |
| <kbd>/</kbd>                        | Focus search input                               |
| <kbd>Escape</kbd>                   | Exit current mode                                |

## Customizing Shortcuts

SQL Pro now supports fully customizable keyboard shortcuts. You can change any shortcut to match your preferred workflow.

### Accessing Shortcut Settings

1. Open **Settings** (<kbd>Cmd/Ctrl</kbd> + <kbd>,</kbd>)
2. Find the **Keyboard Shortcuts** section
3. Click to open the shortcuts customization panel

### Presets

SQL Pro includes several built-in presets to help you get started quickly:

| Preset           | Description                                   |
| ---------------- | --------------------------------------------- |
| **SQL Pro**      | Default shortcuts optimized for database work |
| **VS Code**      | Visual Studio Code-style shortcuts            |
| **Sublime Text** | Sublime Text-style shortcuts                  |
| **Custom**       | Your personalized shortcut configuration      |

### Changing a Shortcut

1. Click on any shortcut in the list
2. Press the new key combination you want to use
3. Click the checkmark to confirm, or X to cancel

### Conflict Detection

SQL Pro automatically detects when two shortcuts use the same key combination. Conflicting shortcuts are highlighted with a warning, allowing you to resolve them.

### Import/Export

You can export your custom shortcuts to a JSON file and import them on other machines:

- **Export**: Click the Export button to download your shortcuts
- **Import**: Click the Import button to load shortcuts from a file

### Reset to Defaults

Click **Reset All** to restore all shortcuts to the SQL Pro defaults.

## Troubleshooting

### Shortcuts Not Working

If keyboard shortcuts aren't responding:

1. **Check focus** - Make sure the correct panel is focused
2. **Check for conflicts** - Browser or OS shortcuts may override app shortcuts
3. **Check Vim mode** - Some shortcuts behave differently in Vim mode
4. **Refresh the page** - Try reloading if shortcuts become unresponsive

### Browser Conflicts

Some shortcuts may conflict with browser defaults:

- <kbd>Cmd/Ctrl</kbd> + <kbd>H</kbd> might open browser history in some browsers
- <kbd>Cmd/Ctrl</kbd> + <kbd>1</kbd>/<kbd>2</kbd> might switch browser tabs

SQL Pro attempts to prevent these defaults when focused, but if you experience conflicts, try using the Command Palette instead.
