Perform find-and-replace across all 15 component CSS files:

**Token migrations (find -> replace):**

1. `var(--text-primary)` -> `var(--foreground)`
2. `var(--text-secondary)` -> `var(--secondary-foreground)`
3. `var(--text-muted)` -> `var(--muted-foreground)`
4. `var(--text-inverse)` -> `var(--primary-foreground)` (for text on primary buttons)
5. `var(--bg-darker)` -> `var(--background)` (or keep as is since it's aliased)
6. `var(--bg-card)` -> `var(--card)`

**Border token standardization:**

- `var(--border-light)` -> `var(--border)` (middle tier)
- Keep `var(--border-subtle)` and `var(--border-medium)` as-is (website may need 4-level granularity)
- Alternatively, map: `--border-medium` -> `var(--border-strong)`

**Approach:**

1. Read each component CSS file
2. Apply the token replacements
3. Write the updated file
4. Verify no syntax errors introduced

**Files to prioritize** (most token usage):

- Hero.css (uses --text-primary, --text-secondary, --text-muted, --bg-darker, --bg-card)
- Features.css
- Pricing.css
- Footer.css

**Files with minimal/no changes** (verify only):

- TopBar.css (no theme tokens)
- SkipLink.css (minimal)

IMPORTANT: Do NOT change:

- Animation/transition tokens (--duration-_, --ease-_)
- Layout tokens (--border-radius-\*, --container-max)
- Raw OKLCH values for gradients/glows (these are intentional one-offs)
