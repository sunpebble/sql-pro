  <action>
1. Create `apps/website/src/hooks/useInView.ts`:
   - Export a `useInView` hook wrapping IntersectionObserver
   - Parameters: `{ threshold?: number, rootMargin?: string, triggerOnce?: boolean }`
   - Return: `{ ref, isInView }` where ref is `RefObject<HTMLElement>`
   - Default options: `threshold: 0.2`, `rootMargin: '0px 0px -50px 0px'`, `triggerOnce: true`
   - Match the proven pattern from Features.tsx (already in codebase)

2. Update `apps/website/src/components/Download.tsx`:
   - Import `useInView` from the new hook
   - Use useInView hook on the download-content wrapper
   - Add `visible` class conditionally based on isInView state
   - Add ref to the download-content div

3. Update `apps/website/src/components/Download.css`:
   - Remove the `animation: fade-up` and `animation-delay` from `.download-content`
   - Change to transition-based approach: `transition: opacity 0.6s var(--ease-out), transform 0.6s var(--ease-out);`
   - Keep initial state: `opacity: 0; transform: translateY(30px);`
   - Add `.download-content.visible` rule: `opacity: 1; transform: translateY(0);`
   - Update reduced motion: `.download-content` should have `opacity: 1; transform: none; transition: none;` (no animation needed)

WHY transition-based: CSS animation fires once on load. Transition + class toggle lets us control when the animation starts via IntersectionObserver.
</action>
